External files

Introduction

External files describe files located outside the build server. They can be used for several purposes:
  • input files for a production build:
    • source files (dita, ditamaps, images, etc.)
    • external files to be included in a produced documentation (for example API)
  • output location (and if necessary output file) for a delivery build

Input external files are copied to their destination without any transformation. They are usually located on remote file systems or repositories (SVN, nexus, RTC), but can also be located in the local RTC repository. They must be gathered in a single directory: all the files in the source directory will be copied to the destination. All the filenames are relative to the load directory, defined in the Jazz Source Control tab of the build definition.

For output external files, the command can be only copy or ftp.
